•
What is clustering?
– Organizing data into so-called clusters such that there is: 
• High intra-cluster similarity
• Low inter-cluster similarity
•
In other words, finding groups of data objects such that
– Objects in a group will be similar (or related) to one another
– And different from (or unrelated to) objects from other groups

5. Clustering Algorithms
5.1. Partitional Clustering: K-means
•
K-means is one of the most popular partitional clustering algorithms
– Requires the number of clusters k to be specified by the user
•
Main process:
1. Initialization: Select k data objects (points) as the initial centroids
2. Repeat
3. Assignment: Form k clusters by assigning objects to the closest centroids
4. Update: Re-compute the centroid for each cluster
5. Until reaching convergence

Centroid: computed as the mean of all data objects in the cluster

•
Closeness to centroid: evaluated using a similarity (distance) measure

•
Convergence: is reached when few (or no) data objects change clusters

5. Clustering Algorithms
5.1. Partitional Clustering: K-means
•
Pros of k-means:
• Intuitive objective function: optimizing intra-cluster similarity 
• Easy to implement and debug
• Relatively efficient: of average O(nki) time and O(n) space complexity, where n is # of 
objects, k is # of clusters, and i is # of iterations
•
Cons of k-means:
• Need to specify k, the number of clusters, in advance
• Applicable only when mean can be defined: what about categorical (or set) data?
• Results depend on the initial centroids: often terminating at local optima
• Not suitable to identify clusters with different sizes and densities
•
Also has problems when clusters have non-convex (non-globular) shapes
• Unable to handle noise or outliers

5. Clustering Algorithms
5.3. Other Approaches: Density-based Clustering
•
Using ɛ and minPts, DBSCAN categorizes objects into:
1. Core Points: A data point p is a core point if its ɛ-neighborhood N(p) contains at 
least minPts: | N(p) | >= minPts, including the point p itself
2. Border Points: A data point q is a border point if N(q) contains less than minPts
data points, but q is reachable from some core point p
- A point q is reachable from p if there is a path p1, ..., pn with p1 = p and pn = q, 
where each pi+1 is directly reachable from pi
- A point p is directly reachable from q if it is in q’s neighborhood (or vice versa)
3. Outliers: A data point o is an outlier if it is neither a core point nor a border point

Wrapping up
•
Clustering consists in finding groups of data objects such that
– Objects in a group will be similar (or related) to one another
– And different from (or unrelated to) objects from other groups
•
Clustering is also known as un-supervised learning
– Versus supervised learning where the groups of data are known
•
Various clustering approaches and algorithms exist
– Partitional, hierarchical, incremental, constrained agglomerative, spectral, density-
based, and fuzzy clustering, among others
– Each with its pros and cons
•
Various measures also exist for evaluating clustering quality
– Internal such as the Dunn index, and external such as Precision and Recall
