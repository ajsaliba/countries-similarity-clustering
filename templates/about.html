ℹ About SIMILICA

Wikipedia Infobox Similarity & Clustering

A research tool for COE 543/743 — Spring 2026.

🔬 Methodology

SIMILICA converts each country's Wikipedia infobox into a rooted labeled tree using a custom schema parser. The tree encodes structural relationships between infobox sections (area, economy, government, population, etc.) and their leaf values.

Tree Edit Distance (TED) is computed using the Zhang-Shasha algorithm — the classical O(n²m) dynamic programming approach that finds the minimum cost sequence of node insertions, deletions, and substitutions to transform one tree into another.

📐 Structural Similarity

Uses TED on the tree structure alone, applying per-section cost weights. The exp_size method normalises by max tree size and applies a size-ratio ceiling (Fix #5).

🔤 Semantic Similarity

Jaccard coefficient over the bag of text tokens extracted from all string leaf values. Captures language, religion, government terminology, and cultural vocabulary overlap.

◎ Clustering Algorithms

  • Agglomerative — UPGMA hierarchical
  • DBSCAN — density-based, flags outliers
  • Spectral — graph Laplacian partitioning
  • K-Medoids — PAM, centre = real country

📊 Visualisations

  • D3.js dendrogram (hierarchical only)
  • 2D MDS scatter plot
  • D3 force-directed graph
  • Leaflet choropleth map

📦 Data

195 countries scraped from Wikipedia infoboxes and cleaned into a canonical JSON schema with sections: area, codes, economy, general, government, population, time.
